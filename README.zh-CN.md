> **依乎天理，批大郤，导大窾。**
> 顺着天然的纹理，劈开大的缝隙，把刀引入骨节间的空腔。
> ——《庄子·养生主》

**你的 AI 在用屠夫的砍刀解复杂系统。教它用厨子的薄刃。**

面对一团纠缠的代码库，或一个含混的「给我做个 X」任务，多数 AI 一头扎进去就开砍——还没看清结构就动文件，先攻耦合最深的核心，处处用同一种速度抡刀。它在砍骨头。刀刃卷了。东西坏了。

**庖丁解牛**（PaoDing JieNiu）封存了两千三百年前庄子笔下那位传奇厨子的答案——他一把刀用十九年仍锋利如新，因为他从不逆纹而切：他找到本来就在那里的缝隙，让刀自己滑过去。

## 问题所在

```
你：「重构这个 4 万行的遗留模块。」

无庖丁的 AI：
  - 打开 3 个文件，立刻动最核心的那个类
  - 碰了全局状态，搞坏 12 处，回滚不了
  - 在加密路径上和在一个 CSS 文件上一样鲁莽地快
  - 「干脆整个重写吧」（新刀又卷——换了新刀，照样砍钝）

有庖丁的 AI：
  - 先读结构：依赖图、数据流、领域边界（依乎天理）
  - 找接缝：纯模块、干净接口、低耦合层（批大郤·导大窾）
  - 在硬骨头处减速——加测试、写小实验（怵然为戒）
  - 小步可逆提交；工具链与信任始终如新（刀刃若新）
```

## 它教给 AI 什么

### 🔪 四底层原则（四底层原则）

| 原则 | 中文 | AI 做什么 |
|------|------|-----------|
| 顺其纹理 | 依乎天理 | 动任何东西之前先读结构 |
| 从缝隙入 | 批大郤·导大窾 | 从接口、纯模块、低耦合层下手 |
| 硬骨处慢 | 怵然为戒·动刀甚微 | 在核心 / 并发 / 安全路径上减速 |
| 养刀常利 | 刀刃若新发于硎 | 小步、可逆、有测试托底——不积债 |

### 🐂 牛体解剖（拆解概念体系）

本 skill 给 AI 一套「往哪里下刀」的词汇：

| 术语 | 中文 | 工程含义 |
|------|------|----------|
| 天理 | 天理 | 系统固有的结构纹理 |
| 大郤 | 大郤（xì） | 一道宽而明显的边界——好下的第一刀 |
| 大窾 | 大窾（kuǎn） | 骨节之间的空腔——刀能自由游走之处 |
| 大軱 | 大軱（gū） | 核心抽象 / 全局状态——切莫硬砍 |
| 族 | 族（zú） | 筋骨盘结的耦合密集区——此处慢下来 |
| 游刃有余 | 游刃有余 | 刀始终留出的可回退余地与余裕 |
| 以神遇 | 以神遇 | 当结构真被吃透，拆解便从容不费力 |

### 📏 拆解四步法（拆解四步法）

```
1. 观 OBSERVE  — 看清这头牛：目录树、依赖图、数据流、领域
2. 寻 SEEK     — 找接缝：第一刀，可回退、可测试
3. 慎 CARE     — 到硬骨头处放慢：测试、小实验、留逃生口
4. 养 SUSTAIN  — 小步提交、常绿、不积债：保持刀刃常新
```

### 以道驭术 — 凭道，不凭蛮力

> 「良庖岁更刀，割也；族庖月更刀，折也。」
> 好厨子一年换一把刀——他在「割」。
> 差厨子一月换一把——他在「砍」。
> ——《庄子·养生主》

差别从不在刀，而在用刀之**道**。庖丁解牛与 **NoPUA** 同源：*以道驭术，不以恐惧驭事；以结构理解取代慌乱猛砍。* 顺纹理而切，正是让刀刃——你的工具、注意力、上下文预算与信任——永远如新的法门。

## 中西合璧

庖丁解牛不取代你的工程训练，它**磨利**它。

| 西方实践 | + 庖丁解牛 | = 完整 |
|----------|-----------|--------|
| 「先跑通，再重构」 | 先依乎天理 | 切之前先懂——少返工 |
| 找接缝（Michael Feathers） | 批大郤·导大窾 | 关于接缝**在哪里**的一套哲学 |
| 「快速行动，打破常规」 | 在硬骨头处怵然为戒 | 空处求快，硬骨头处求慢 |
| 童子军规则 | 刀刃若新发于硎 | 磨利的是整条工具链与信任，不只一个文件 |
