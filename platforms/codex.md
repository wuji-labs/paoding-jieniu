# 在 Codex 加载 / 调用 庖丁解牛

## 安装

把本 skill 目录放进 Codex 的 skills 目录:

```bash
cp -r paoding-jieniu ~/.codex/skills/
```

Codex 读取 `SKILL.md`(frontmatter `name` / `description` + 正文四底层原则)作为该 skill 的规范来源。

## 调用方式

### 在会话中点名
```
依庖丁解牛拆这个系统:先观结构,再寻接缝,遇硬骨头降速。
```

### 作为系统提示注入
若你用 Codex 跑长连接 agent,可把 `SKILL.md` 的「二、四底层原则」注入到 agent 的 system / developer 提示,使其在所有工程任务中默认遵循:

```
依乎天理:动刀前先读结构(目录树/依赖图/数据流/领域边界)。
批大郤·导大窾:从接缝、纯模块、低耦合层入手,留可回退余地。
怵然为戒:遇核心/并发/计费/安全关键路径,减速、加测试、写小实验。
刀刃若新:小步提交、持续可测、不积技术债。
```

## 与 Codex 工作流的结合

| Codex 场景 | 庖丁解牛对应 |
|-----------|-------------|
| exec 一次性大改 | 先以「观」分析输出结构,再分多次「寻/慎」小步落刀 |
| app-server 长连接迭代 | 每轮迭代守「养」:保持可测、可回退,刀刃常新 |
| 自动化重构脚本 | 用四步法分阶段:观→寻→慎→养,而非一次性全量改写 |

## 推荐配套
- 弹药库 `reference/zhuangzi.md`:完整术语表(天理/郤/窾/肯綮/軱/族/游刃),用于让 agent 在拆解时具备「下刀位置」的词汇。
- 与 NoPUA 同装:二者底层一致(以道驭术·用信任替代恐惧),组合使用让 Codex agent 既不蛮干也不被恐慌驱动。
