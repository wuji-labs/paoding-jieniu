# 在 Cursor 加载 / 调用 庖丁解牛

## 安装(Project Rules)

Cursor 通过 `.cursor/rules/` 下的 `.mdc` 规则文件注入 skill。把本 skill 的四底层原则落成一条规则:

新建 `.cursor/rules/paoding-jieniu.mdc`:

```mdc
---
description: 庖丁解牛 — 复杂系统拆解方法论(依乎天理·批大郤·导大窾·游刃有余)
globs:
alwaysApply: false
---

# 庖丁解牛 PaoDing JieNiu — The Ox-Carving Way

面对庞大/纠缠的代码库或含混的大任务时,遵循四底层原则:

1. 依乎天理 — 动刀前先读结构:目录树、依赖图、数据流、领域边界,建心智模型。禁止未看清结构就改文件。
2. 批大郤·导大窾 — 从系统本有的接缝下手:接口、纯模块、低耦合层。每一刀留可回退余地(游刃有余)。
3. 怵然为戒 — 遇硬骨头(核心抽象/全局状态/并发/计费/安全)减速:加测试、写小实验、留退路,宁慢十倍。
4. 刀刃若新 — 小步提交、持续可测、不积技术债,保持工具与信任常新。

拆解四步法:观(读结构)→ 寻(找接缝)→ 慎(硬骨头降速)→ 养(小步可测)。
```

## 调用方式

### 自动应用
把 `alwaysApply` 设为 `true`,则所有 Cursor 对话默认载入庖丁解牛原则;适合重构 / 遗留系统 / 架构类项目。

### 按需引用
保持 `alwaysApply: false`,在 Composer / Chat 中用 `@paoding-jieniu` 显式引用该规则,或直接写:

```
@paoding-jieniu 用四步法拆解 src/legacy/
```

### 路径触发
若想只在特定目录生效,给规则加 `globs`(如 `src/legacy/**`),进入这些文件时自动激活。

## 与 Cursor 工作流的结合

| Cursor 动作 | 庖丁解牛对应 |
|------------|-------------|
| Composer 多文件改动 | 先「观」全局,再分步「寻/慎」,避免一次性大改不可回退 |
| Chat 解释陌生代码库 | 用「三重境界」引导:从见全牛 → 看穿结构 → 定下刀点 |
| Agent 自动重构 | 守「养」:每步保持可测、可回退,刀刃常新 |
